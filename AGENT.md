# AGENT.md

## Mission

Maintain the JLPT study materials, browser learning app, and optional sync worker as one coherent project. Preserve learner progress and generated-data reproducibility above all else.

## Essential commands

```bash
npm run dev          # serve web/ at http://localhost:8000
npm test             # JavaScript, Python, and vocabulary-data validation
npm run build:data   # regenerate every browser JSON deck
```

PDF books are generated separately:

```bash
python3 build_vocab_pdf.py n5
python3 build_grammar_pdf.py n5
```

Replace `n5` with the required JLPT level.

## Repository map

- `data/`: source vocabulary and grammar chunks plus raw exercise items.
- `web/data/`: validated browser decks generated by the root Python builders.
- `web/js/app.js`: application coordinator and mode/deck routing.
- `web/js/store.js`, `srs.js`, `progress.js`: persisted state, scheduling, and daily progress.
- `web/js/modes/`: vocabulary games, grammar exercises, and reading launcher.
- `worker/`: Google-authenticated Cloudflare Worker/KV synchronization.
- `tests/`: dependency-free Node tests and Python validator tests.
- `TODO.md`: current learner/project handoff; history belongs in Git.

## Working rules

- Keep persisted localStorage and remote state backward compatible.
- Never change stable card IDs by hand; builders derive them from content.
- Edit source data first, then regenerate and validate derived `web/data` files.
- Keep the browser app dependency-free unless a dependency brings a clear operational benefit.
- Add pure tests for scheduling, state merging, grading, and validation changes.
- Run `npm test` and `git diff --check` before committing.
- Bump the cache name in `web/sw.js` whenever a deployed shell asset changes.
- Preserve unrelated user changes in a dirty working tree.

## Handoff

Read the `Current` section of `TODO.md` and recent Git history before changing the project. Update `TODO.md` when the active state or next action materially changes.
