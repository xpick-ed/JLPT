# CLAUDE.md

> Two-machine sync & handoff protocol lives in the global `~/.claude/CLAUDE.md`.
> In short: at start `git fetch` + `git pull --rebase` (if a remote exists) + read `TODO.md`;
> at end update `TODO.md`, commit, and push (if a remote exists). Work on `main` only.

## Project info

- Stack: dependency-free HTML/CSS/ES modules, Python data/PDF builders, Cloudflare Worker + KV for optional Google-auth sync.
- Setup / install: Python 3 and Node.js 20+; PDF generation additionally needs ReportLab and the fonts described in the builder modules. The web app itself has no npm dependencies.
- Build: `npm run build:data` regenerates all browser JSON decks. PDF books are built with `python3 build_vocab_pdf.py n5` or `python3 build_grammar_pdf.py n5` (replace the level as needed).
- Test: `npm test` runs JavaScript tests, Python unit tests, and the vocabulary data-integrity build.
- Run: `npm run dev`, then open `http://localhost:8000`.

## Structure notes

- `data/`: source vocabulary/grammar chunks plus raw cloze/order exercise items.
- `web/data/`: validated, deduplicated browser decks generated by `build_*.py`.
- `web/js/app.js`: application coordinator; `store.js`, `srs.js`, and `progress.js` own persisted learning state.
- `web/js/modes/`: four vocabulary modes, two grammar modes, and the reading launcher.
- `worker/`: Google ID-token exchange and per-user Cloudflare KV synchronization.
- `docs/superpowers/`: historical feature specs and implementation plans.
- Root PDFs/CSVs are generated study and Anki deliverables and are intentionally tracked.

## Conventions

- Keep the web app dependency-free unless a dependency has a clear operational benefit.
- Persisted state changes must remain backward compatible with older localStorage and remote blobs.
- Stable card IDs are SHA-1 prefixes derived by the builders; do not assign IDs manually.
- Add pure unit tests for scheduling, merging, grading, and data validation logic.
- Bump the cache name in `web/sw.js` whenever a deployed shell asset changes.
